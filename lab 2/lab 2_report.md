Отчет по лабораторной работе №2 "Исследование Cloud Run". 

Целью работы было ознакомиться с работой сервиса Cloud Run в Google Cloud, научиться разворачивать сервисы, управлять ревизиями, изменять конфигурацию и распределять трафик между версиями. В ходе выполнения работы я развернула сервис Cloud Run из дефолтного образа Hello World, выбрав минимальную конфигурацию ресурсов. Для этого я использовала облачную консоль и команду gcloud run deploy с указанием региона europe-west1, памяти 256Mi и разрешением неаутентифицированных вызовов. После успешного развертывания я получила URL сервиса https://lab2-hello-vvalieva-307056602443.europe-west1.run.app и перешла по нему в браузере, где увидела стандартное приветственное сообщение Hello World, что подтвердило корректную работу сервиса. Затем я перешла в раздел логирования и анализа метрик сервиса. Во вкладке LOGS я изучила записи о запуске контейнера, где было указано, что контейнер успешно запущен и слушает HTTP запросы на порту 8080. Во вкладке METRICS я проанализировала графики количества запросов, где наблюдался небольшой пик в момент моего перехода по ссылке, а также графики потребления CPU и памяти, которые находились на минимальном уровне. Далее я провела эксперимент с изменением порта. В настройках сервиса я создала новую ревизию, изменив параметр Container Port с 8080 на 8090. После развертывания новой ревизии я проанализировала логи и обнаружила, что контейнер успешно перезапустился и теперь слушает порт 8090, о чем свидетельствовала запись в логах: Hello from Cloud Run! The container started successfully and is listening for HTTP requests on port 8090. При этом сам сервис продолжал корректно отвечать по тому же URL, поскольку используемый образ контейнера оказался способен динамически подстраиваться под указанный порт. Затем я перешла к управлению трафиком между версиями. В интерфейсе Cloud Run я открыла окно Manage Traffic и выбрала опцию распределения трафика между несколькими ревизиями. Я установила следующие пропорции: на ревизию lab2-hello-vvalieva-00001-kkl, работающую на порту 8080, направила 50 процентов трафика, а на ревизию lab2-hello-vvalieva-00002-p2r, работающую на порту 8090, также направила 50 процентов трафика. После сохранения настроек я несколько раз обновила страницу сервиса в браузере и убедилась, что сервис продолжает работать корректно, а запросы распределяются между двумя версиями.

Вывод: в ходе лабораторной работы я освоила основные возможности сервиса Cloud Run. Я научилась развертывать контейнерные приложения из готового образа, анализировать логи и метрики для мониторинга работы сервиса, создавать новые ревизии с измененными параметрами конфигурации, управлять распределением трафика между несколькими версиями сервиса.
